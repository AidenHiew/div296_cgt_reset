# Division 296 Model v2.0.0 UX Pass Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v2.0.0 of the Division 296 Cost Base Reset Model — a UX/layout/formatting refresh of the Python-built `.xlsx` workbook, with v1.0.0 frozen as a reference build.

**Architecture:** Source of truth is the Python builder under `src/div296/`. Each tab is a module under `src/div296/tabs/`. Styles centralised in `src/div296/styles.py`. Build entrypoint `python -m div296.build` produces `dist/Division_296_Model_v<version>.xlsx`. Changes are additive (new palette tokens, new layout elements) and surgical (rename header, shift row/column constants); no logic, formula, or calculation changes.

**Tech Stack:** Python 3.11+, openpyxl (workbook construction), pytest (test suite), GitHub Actions (CI). Existing patterns: dataclass-based config in `assumptions.py`, named ranges in `named_ranges.py`, per-tab modules export a `build(wb)` function called from `build.py`.

**Source spec:** [`docs/superpowers/specs/2026-05-22-v2-ux-pass-design.md`](../specs/2026-05-22-v2-ux-pass-design.md)

## Hard constraints (read before starting)

These are from `~/.claude/CLAUDE.md` and the project handoff — violations are blockers, not style issues:

- **No `git push`** unless the user has typed the sentinel phrase `push this branch now`. CI runs on push, so propose-then-wait.
- **No merge to main** — branch-protected; merges go via PR with user approval.
- **No weakening of tests / lints / CI / hooks.** If a test fails, fix the test or fix the bug — don't relax the test.
- **One commit per logical change** (this plan is structured around that).
- v1.0.0 stays frozen. Do not edit files in a way that would change the v1.0.0 build output. The branch `v2/ux-pass` is the workspace.
- Use `--no-validate` when invoking `python -m div296.build` locally — the recalc gate OOMs the `formulas` package (see handoff §"OOM gotcha").
- Run fast tests with `pytest -m "not slow"`. Never run `pytest -n auto` on the user's machine.
